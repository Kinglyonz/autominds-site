Enterprise-Grade Safety Controls

AMI includes fail-safe mechanisms to prevent unauthorized or destructive actions

Dry Run Mode

Test all operations without making changes. AMI simulates actions and reports what would happen. Enabled by default on first deployment.

Environment variable: DRY_RUN_MODE=true

Budget Caps

Set maximum spending limits per month. AMI refuses to execute operations that would exceed budget. Alerts sent 24 hours before cap is reached.

Default: $100/month, configurable via BUDGET_CAP_USD

Human Approval Workflows

Require Slack or email approval for destructive operations. AMI sends approval request with blast radius assessment. Auto-expires after 1 hour.

Supports Slack webhooks, PagerDuty, ServiceNow

Blast Radius Classification

Every operation is classified (NONE, SINGLE, SERVICE, CROSS_SERVICE). High-impact actions require elevated approval even if budget allows.

Algorithm considers resource dependencies and downtime risk

Rollback Mechanism

AMI records state before making changes. If operation fails or user requests rollback, original state is restored automatically.

State stored in DynamoDB with 90-day retention

Complete Audit Trail

Every action logged to DynamoDB with timestamp, operator, action, result, and Claude API request/response. Tamper-proof with cryptographic signatures.

SOC2 Type II compliant logging format
